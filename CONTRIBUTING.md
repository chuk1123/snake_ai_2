# Contributing

Thanks for your interest in contributing!

- Fork the repository and create a branch for your change.
- Make small, focused commits with descriptive messages.
- Include tests or example inputs when adding functionality.
- Open a pull request describing your change.

Code style
- Follow the project's existing code style where possible.
- For small scripts, prefer simple, readable code with comments.

Reporting issues
- File issues for bugs or feature requests and include reproduction steps.

Maintainer notes
- Please run the test (if any) and ensure the linter passes before merging.
